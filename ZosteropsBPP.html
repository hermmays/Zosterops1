BPP and GDI based species delimitation

Goals

What follows is our workflow for the Bayesian Phylogenetics and Phylogeography (BP&P or hereafter just BPP) and genealogical diversity index (GDI) analyses for East Asian Zosterops. Using the multispecies coalescent (Flouri et al. 2019), the BPP software package v4.7.0 (Flouri et al. 2018), and the GDI heuristic species delimitation methods outlined in Leaché et al. (Leaché et al. 2018) this analysis attempts to estimate 𝜃 and test species hypotheses against the eight locus Sanger sequencing dataset for East Asian Zosterops (see Sanger sequencing summary statistics and trees section).

Getting started

We are using R version 4.4.2 and RStudio version 2024.09.1 to illustrate the guide trees in for BPP, calculate GDI, and plot 𝜃 and GDI. Quarto version 1.6.39 for Mac OS was used to document the workflow. The analysis was conducted using BPP v4.7.0. The BPP manual is an excellent introduction to these analyses.

Software and packages

  • BPP v4.7.0 (Flouri et al. 2018) A newer version of BPP is available but this version appears to work fine. To replicate this analysis, run on an install of BPP v4.7.0 available here. Documentation for BPP is available here.

  • ape (Paradis and Schliep 2019) The ape package was used to visualize the guide trees used in the BPP analyses.

Load the ape library before proceeding.

Data files

Sequence alignments

The data used for this analysis will be restricted to those Sanger sequences from Zosterops japonicus, Z. meyeni, Z. montanus, and Z. simplex. The sequence file is in phylip format.

To illustrate the data format an excerpt of the sequence file ZosteropsBPP_JMMSseq.txt is shown below (ellipsis indicates where additional sequences from the original file have been left out). One alignment of individual sequences follows the other sequentially in a single file. A space followed by two numbers indicate the beginning of a locus. First is the number of individual sequences for that locus. Remember that for those loci from phased nuclear coding DNA (nuDNA) there will be two sequences for each individual with the individual identifying code followed by a and b to denote the two phased alleles (see Sanger sequencing summary statistics and trees section). The following number is the length of the sequence alignment in base pairs (bp). Each sequence will lead with the sample, an a for the mitochondrial DNA (mtDNA) loci or either a or b for the phased cdDNA loci, and underscore, the locus name (e.g. COI, ND2, B3GNT9), a ^ symbol, and finally the individual sample name (minus the allelic designations, a and/or b). The individual sample name following the ^ symbol should match the individual names used to map taxon names (i.e. the imap file).

Taxon maps and guide trees

The final data file needed for these analyses is an imap file saved in txt format that maps the taxon and/or population designations to individual samples. Individual samples in the imap file should match those in the phylip formatted sequence file. Four imap files will be needed for the analysis.

  • ZosteropsBPPtip_imap.txt

  • ZosteropsBPPmid1_imap.txt

  • ZosteropsBPPmid2_imap.txt

  • ZosteropsBPProot_imap.txt

Codes in these files are designated for each taxon as follows.

  • Jp - Z. japonicus

  • Me - Z. meyeni

  • MoN - Z. montanus Northern Philippines including subspecies halconensis (Mindoro) and whiteheadi (Luzon)

  • MoS - Z. montanus Southern Philippines, Palawan, and Indonesia including subspecies Z. montanus diuatae (Mindanao), Z. montanus vulcani (Mindanao), Z. montanus pectoralis (Negros), Z. montanus parkesi (Palawan), Z. montanus montanus (Indonesia), and an unnamed Z. montanus population on the Philippine island of Panay.

  • Si - Z. simplex including populations on the island of Taiwan and Mainland Asia.

BPP control files

Control files were created to run BPP analyses across a variety of models and prior distributions.

BPP A10 control files

The BPP analyses designated the A runs conduct a species delimitation analysis on a fixed species tree. The species tree used for the A10 runs is the TIP tree with each taxon as a separate branch (Z. japonicus, Z. meyeni, Z. montanus from the Northern Philippines, Z. montanus from the Southern Philippines and Indonesia, and Z. simplex, see above description of associated tree files). Below is the ctl file for the first A series BPP run (A1). Several parameters on the control (ctl) files are edited to execute different runs including the \(β_θ\) and \(β_τ\) parameters for the inverse 𝛤 prior distribution, and the species delimitation model (0 or 1). The \(β_θ\) and \(β_τ\) parameters for the inverse 𝛤 prior distribution are edited at the thetaprior and tauprior lines, each of which is specified by two digits for 𝛼 and 𝛽, but only the 𝛽 value will be changed across runs. A list of the ctl files and the key parameters for each A series run are shown in the BPP A10 runs table in the A10 results section below. Note that beginning a line in the ctl file with an asterisk causes BPP to ignore that line. Adding and deleting asterisks in front of particular lines of commands is a convenient way to add or delete particular parameters.

BPP A00 control files

The BPP analyses designated the B, C, D, and E runs work from a fixed species tree without the BPP species delimitation analysis. For the B runs the \(β_θ\) and \(β_τ\) parameters for the inverse 𝛤 prior distribution are 0.001 and 0.004 respectively. For the C runs the \(β_θ\) and \(β_τ\) parameters are 0.002 and 0.004, for the D runs these values are 0.001 and 0.0005, and for the E runs these values are 0.002 and 0.0005. Running BPP independently for different, but reasonable, distributions of 𝜃 and 𝜏 is used to assess the sensitivity of our estimates to the priors. Note that for all the A00 runs (run series B-E) the speciesdelimitation selections are omitted by being preceded by an asterisk, as is the speciesmodelprior selection. Three replicate runs are done for each combination of 𝜃 and 𝜏 priors and each species tree (TIP, MID1, MID2, and ROOT, see above). Starting each ctl file with seed = -1 creates a different random number seed for each replicate run. Independent runs for each species tree will be used to estimate 𝜃 and 𝜏 which will in turn be used to estimate the GDI for each branch (see description of GDI below). Below is an example of a B run control file. A list of the ctl files and the key parameters for each set of runs in the B-E series run are shown in the BPP A00 runs table in the A00 results section below.

seed =  -1
seqfile = ZosteropsBPP_JMMSseq.txt
Imapfile = ZosteropsBPPtip_imap.txt
outfile = ZosteropsBPP_A00_RUNB1.out.txt
mcmcfile = ZosteropsBPP_A00_RUNB1.mcmc.txt

Genealogical diversity index (GDI)

\(2\tau/\theta_A\) is the population divergence time for population \(A\) in coalescent units. One coalescent unit is equal to \(2N_A\) generations where \(N_A\) is the effective population size of \(A\). \(e^{(-2\tau/\theta_A)}\) is the probability that two sequences (\(a_1\) and \(a_2\)) from population \(A\) fail to coalesce before the divergence (𝜏) of \(A\) from its sister lineage. The GDI is the complementary probability of \(e^{(-2\tau/\theta_A)}\) and reflects the probability that \(a_1\) and \(a_2\) coalesce prior to (going backwards in time) the divergence between population \(A\) and its sister lineage. GDI therefore may be calculated from 𝜃 and 𝜏 using the following equation (Jackson et al. 2016; Leaché et al. 2018).

\[ GDI = 1 - e^{(-2\tau/\theta_A)} \]

The BPP A00 analyses will estimate 𝜃 and 𝜏 along each branch in a given tree and we then use those values to calculate the GDI for each branch.

Running BPP

Upon installing BPP v4.7.0 a directory will be created containing a bin folder with the BPP application (on Mac OS that directory is bpp-4.7.0-macos-aarch64). To execute a BPP run copy the following needed files to this directory (in this example we will be running the B1 run, an A00 analysis with ctl file ZosteropsA00_RUNB1.bpp.ctl).

  • ZosteropsA00_RUNB1.bpp.ctl

  • ZosteropsBPP_JMMSseq.txt

  • ZosteropsBPP_heredity.txt

  • ZosteropsBPPtip_imap.txt

The only files that change between runs is the ctl file and the imap file.

Navigate to the BPP directory with the bin folder containing the bpp program.

In the terminal run the following command (make sure the needed files are in the working directory).

bin/bpp --cfile ZosteropsA00_RUNB1.bpp.ctl

Upon completion multiple files will be generated by this analysis.

  • FigTree.tre This is a tree file generated with 𝜃, 𝜏, and 95% highest posterior density (HPD) for 𝜏 as branch lengths based on the tree topology given in the analysis.

  • Eight sample.txt files, one for each locus in the sequence file (locus.[1-8].sample.txt).

  • SeedUsed A text file recording the random number seed used for the run.

  • ZosteropsBPP_A00_RUNB1.mcmc.txt A text file containing the results from the MCMC chain.

  • ZosteropsBPP_A00_RUNB1.out.txt A text file summarizing the output of the run (for A10 runs this will include whether the analysis supported a split at each node along with the posterior probability of a split). For each run the name of the mcmc and out files will be specified by the ctl file.

Repeat this process for each ctl file and for the A00 runs replicate each run three times. Once each run is complete, copy and organize the results to an appropriate directory.
